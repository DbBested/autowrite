# Revision Plan Pass Rules

## Scope: What This Pass Does

- Read `diagnosis.md` and create a structured `plan.md` referencing specific diagnosis findings — every planned change must cite the diagnosis item it addresses (e.g., "Structure pass: move thesis from paragraph 4 to paragraph 1 — diagnosis item: buried lede")
- Organize planned changes by pass — which stage will address which diagnosis finding
- Note which diagnosis findings are out of scope for any single pass, or require author judgment
- Flag voice preservation risks at the planning stage (e.g., "clarity pass on paragraph 3 may flatten fragment-heavy style — preserve fragments explicitly")
- Produce `plan.md` as the sole output of this pass

## DO NOT Touch (Out of Scope)

- The draft text — this pass produces `plan.md` only; no draft modifications
- Do not begin executing revisions — planning and execution are strictly separated
- Do not add new diagnosis findings not present in `diagnosis.md`
- Do not plan changes that would introduce new claims, evidence, or citations not in the original draft

## Voice Preservation

- The plan must explicitly flag any pass where voice drift risk is elevated
- For each pass that touches prose (clarity, tone, concision), note the specific author voice patterns from `diagnosis.md` that must survive that pass
- Example: "Concision pass — preserve author's sentence fragments in introductory paragraph; cut only mechanical filler words"

## Factual Integrity

- The revision plan must not include any changes that would add new claims, invent supporting evidence, or alter existing citations
- If diagnosis flagged an evidence gap, the plan should note it as "flag for author — do not fabricate evidence" rather than planning a fill-in
- Plans that would require invented content must be explicitly rejected: "Out of scope: adding evidence for claim in paragraph 2 — requires author input"
