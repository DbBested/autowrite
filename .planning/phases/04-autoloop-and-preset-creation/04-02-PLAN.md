<objective>
Create the /autoloop skill that runs a mutation-eval self-improvement cycle on preset files, accepting changes only when scores measurably improve without critical regressions.

Purpose: This is the system's ability to improve itself. A preset that was hand-tuned in Phase 1 can be iteratively refined by proposing atomic mutations, measuring their effect via the eval system, and keeping only improvements. The holdout set prevents Goodhart's Law overfitting.

Output: `.claude/skills/autoloop/SKILL.md` — a complete skill definition for the mutation-eval loop, plus `autoloop/holdout/` directory setup.
</objective>

    Create `.claude/skills/autoloop/SKILL.md` implementing the full mutation-eval self-improvement cycle. Follow the step-by-step SKILL.md structure from improve/SKILL.md and eval/SKILL.md.

    **Frontmatter:**
    - name: autoloop
    - description: Run a mutation-eval self-improvement cycle on a preset. Proposes atomic mutations, measures impact via /eval, keeps only improvements. Usage: /autoloop --target presets/blog-post.json --iterations 5 --reference-draft @drafts/ref.md

    **Invocation syntax (per CONTEXT.md locked decision):**
    ```
    /autoloop --target presets/<id>.json --iterations <N> --reference-draft @<file.md>
    ```
    All three flags are required. Reference draft is required per RESEARCH.md open question resolution — fail loudly if not provided.

    **Step 1: Argument Parsing and Validation**
    - Validate `--target` is a `presets/*.json` file that exists. STOP if not found. STOP if target is not in `presets/` directory (prevents mutation of skills, CLAUDE.md, or other files — per RESEARCH.md Pitfall 4). Error: "Autoloop targets are restricted to presets/*.json files. Received: <path>"
    - Validate `--iterations` is a positive integer (1-20). Default: 5. STOP if > 20: "Maximum 20 iterations to prevent context exhaustion."
    - Validate `--reference-draft` file exists. STOP if not found: "Reference draft not found: <path>. The autoloop requires a fixed reference draft to produce comparable before/after scores."
    - Check `autoloop/holdout/` directory. Count `.md` files. If zero: display WARNING: "autoloop/holdout/ is empty. Running without Goodhart's Law protection. The loop will not check for overfitting. Recommended: place 1-3 holdout texts in autoloop/holdout/ and rerun." Ask user to confirm continuation or provide holdout texts.

    **Step 2: Create Run Directory (per RESEARCH.md Pattern 1)**
    ```
    autoloop/runs/<timestamp>/
    ├── backup/              (original target file before any mutation)
    ├── reference-outputs/   (writing engine outputs: before and after)
    ├── evals/               (eval.json files: before and after)
    └── mutations.jsonl      (append-only mutation log)
    ```
    - Generate timestamp: `date +"%Y-%m-%dT%H-%M-%S"`
    - Create all subdirectories
    - Copy `--target` file to `backup/` directory immediately

    **Step 3: Run Baseline (LOOP-02)**
    This step establishes the pre-mutation score baseline.
    1. Run `/improve @<reference-draft> --preset <target-preset-id>` to generate a revised text using the CURRENT (unmutated) preset
    2. Copy the output to `reference-outputs/<draft-slug>-before.md`
    3. Run `/eval @<reference-output> --preset <target-preset-id>` to score the baseline output
    4. Copy eval.json to `evals/<draft-slug>-before.json`
    5. Record baseline scores: read `aggregate_score` and all criterion scores from the eval.json

    **Step 4: Iteration Loop (LOOP-01, LOOP-03)**
    For iteration = 1 to N:

    **Step 4a: Select Mutation Target (per RESEARCH.md Pattern 7 — weakest-criterion-first)**
    1. Read the most recent eval scores (baseline for iteration 1, or post-mutation from previous accepted iteration)
    2. Find the criterion with the lowest score
    3. Map that criterion to the most relevant preset field:
       - Low `voice_preservation` → mutate `voiceBehaviors` array or `voice.rhetoricalStyle`
       - Low `clarity` → mutate stage order or `rubric.criteria[clarity].description`
       - Low `structure` → mutate `stages` ordering or `structure.expectedSections`
       - Low `novelty` → mutate `rubric.criteria[novelty].description`
       - Low `concision` → mutate `stages` (add/reorder concision pass)
       - Low `audience_fit` → mutate `voice.formality` or `voiceBehaviors`
       - Low `factual_integrity` → mutate `rubric.criteria[factual_integrity].description` (tighten anchoring)
    4. If the weakest criterion was the target of the last 2 mutations: round-robin to next-weakest
    5. Propose exactly ONE change to that field with a written rationale explaining why this mutation should improve the score

    **Step 4b: Apply Mutation**
    1. Read the current target preset file
    2. Apply the proposed change (edit exactly one field/section)
    3. Write the mutated preset to `presets/<id>.json`
    4. Run `bash scripts/validate-preset.sh presets/<id>.json` — if validation fails, restore from backup, log rejection with reason "validation_failed", continue to next iteration
    5. Bump the preset's `version` field (patch increment: 1.0.0 -> 1.0.1 -> 1.0.2 etc.)

    **Step 4c: Run Post-Mutation Eval**
    1. Run `/improve @<reference-draft> --preset <target-preset-id>` with the MUTATED preset
    2. Copy output to `reference-outputs/<draft-slug>-after-iter-N.md`
    3. Run `/eval @<reference-output> --preset <target-preset-id>` to score the post-mutation output
    4. Copy eval.json to `evals/<draft-slug>-after-iter-N.json`

    **Step 4d: Acceptance Decision (LOOP-04 — per RESEARCH.md Pattern 3)**
    ALL four conditions must be true for acceptance:
    1. `scores_after.aggregate > scores_before.aggregate` (any improvement)
    2. `scores_after.factual_integrity >= 6` (critical floor)
    3. `scores_after.voice_preservation >= 6` (critical floor)
    4. For EACH criterion C: `scores_after[C] >= scores_before[C] - 2` (no criterion drops > 2 points)

    If ALL conditions true: ACCEPT — keep the mutated preset, update "current scores" to post-mutation scores for next iteration.
    If ANY condition false: REJECT — restore the original preset from `backup/`, log the rejection reason.

    On acceptance: copy the accepted mutation diff to `autoloop/accepted/<timestamp>-iter-N.patch`.

    **Step 4e: Log Mutation (LOOP-05 — per RESEARCH.md Pattern 2)**
    Append one JSONL line to `mutations.jsonl`:
    ```json
    {
      "iteration": N,
      "timestamp": "<ISO 8601>",
      "target_asset": "presets/<id>.json",
      "mutated_field": "<field.path>",
      "mutation_rationale": "<why this mutation should improve scores>",
      "diff": "<unified diff of the change>",
      "scores_before": { "novelty": N, "clarity": N, ... "aggregate": N.NN },
      "scores_after": { "novelty": N, "clarity": N, ... "aggregate": N.NN },
      "decision": "accepted|rejected",
      "decision_reason": "<specific reason citing which conditions passed/failed>"
    }
    ```
    This is append-only. Never read-parse-rewrite the file. Only append.

    **Step 4f: Holdout Check (LOOP-06 — per RESEARCH.md Pattern 4)**
    Every 3 iterations (iterations 3, 6, 9, 12, 15, 18) AND at the final iteration:
    1. Read all `.md` files in `autoloop/holdout/`
    2. For each holdout text: run `/improve` with current preset, then `/eval` on the output
    3. Compute holdout aggregate score (average of all holdout eval aggregate scores)
    4. Compute loop aggregate score (average of `scores_after.aggregate` from last 3 mutations.jsonl entries)
    5. If `|holdout_aggregate - loop_aggregate| > 1.0`:
       - HALT the loop immediately
       - Log halt to mutations.jsonl: `{"iteration": N, "decision": "HALTED", "decision_reason": "Holdout divergence: holdout_aggregate=X.XX, loop_aggregate=Y.YY, delta=Z.ZZ > 1.0 threshold"}`
       - Restore preset from backup
       - Display: "AUTOLOOP HALTED: Holdout divergence detected (delta: Z.ZZ points). This indicates the loop is overfitting to the reference draft. The original preset has been restored."
    6. If holdout is within threshold: continue to next iteration

    **Step 5: Completion Summary**
    After all iterations complete (or loop halts), display:
    ```
    Autoloop complete.

    Target:          presets/<id>.json
    Iterations:      N attempted, M accepted, K rejected [, HALTED at iteration H]
    Reference draft: <path>
    Holdout texts:   N files checked

    Score trajectory:
      Baseline:   X.XX
      Final:      Y.YY  (+/- delta)

    Accepted mutations:
      Iter 1: mutated voiceBehaviors[3] — aggregate 7.15 -> 7.35 (+0.20)
      Iter 4: mutated stages order — aggregate 7.35 -> 7.50 (+0.15)

    Mutation log: autoloop/runs/<timestamp>/mutations.jsonl
    ```

    **Edge Case Table:**
    | Situation | Action |
    |-----------|--------|
    | --target not in presets/ | STOP — "Autoloop targets restricted to presets/*.json" |
    | --target file not found | STOP — "Target preset not found: <path>" |
    | --reference-draft not found | STOP — "Reference draft not found: <path>" |
    | --iterations > 20 | STOP — "Maximum 20 iterations" |
    | autoloop/holdout/ empty | WARN, ask user to confirm or provide holdout texts |
    | validate-preset.sh fails after mutation | Restore backup, log rejection, continue |
    | Holdout divergence > 1.0 | HALT loop, restore backup, display warning |
    | All iterations rejected | Display summary, restore backup, suggest smaller mutations |
    | /improve or /eval fails mid-loop | STOP loop, restore backup, display error |
