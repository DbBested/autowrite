    **Per D-03, D-09, D-10 (PRES-01, PRES-05):**

    1. Create `presets/preset-schema.json` as a JSON Schema Draft 7 file defining the preset format. The schema must require these top-level properties:
       - `id` (string) - unique identifier e.g. "blog-post"
       - `name` (string) - display name e.g. "Blog Post"
       - `description` (string) - one-line description of the form
       - `form` (string) - form identifier matching id
       - `version` (string, pattern "^\\d+\\.\\d+\\.\\d+$") - semver
       - `goals` (array of strings, minItems 1)
       - `stages` (array of strings, minItems 3) - pass sequence, must start with "diagnose"
       - `voice` (object, required properties: tone, formality, sentenceLength, paragraphStyle, rhetoricalStyle — all strings)
       - `structure` (object, required properties: expectedSections [array of strings], sectionOrder [string], paragraphPatterns [array of strings], introStyle [string], endingStyle [string])
       - `rubric` (object, required properties: criteria [array of objects each with name/description/weight], passing_threshold [number], critical_criteria [array of strings containing at minimum "factual_integrity" and "voice_preservation"])
       - `constraints` (object, required properties: no_citation_invention [boolean, const true], no_stance_shift [boolean, const true], aggressive_rewrite_requires_explicit_request [boolean])
       - `transformations` (object, required properties: preserveVoice [boolean], allowMajorRestructure [boolean], prioritizeClarity [boolean], prioritizePersuasion [boolean], prioritizeConcision [boolean])
       - `examples` (array, may be empty)
       Set `"$schema": "http://json-schema.org/draft-07/schema#"`, `"type": "object"`, `"additionalProperties": false`.
       The rubric.criteria items schema: `{"type": "object", "required": ["name", "description", "weight"], "properties": {"name": {"type": "string"}, "description": {"type": "string"}, "weight": {"type": "number", "minimum": 0, "maximum": 1}}}`.

    2. Create `scripts/validate-preset.sh` as an executable bash script using jq 1.6+:
       - Shebang: `#!/usr/bin/env bash`
       - `set -euo pipefail`
       - Accept one argument: path to preset JSON file. Print usage and exit 1 if missing.
       - Verify file exists and is valid JSON (jq . "$FILE" > /dev/null).
       - Define a `check_field` function that takes field_name, jq_expression, expected_type_description. If `jq -e "$jq_expr" "$FILE"` fails, print `ERROR: Missing or invalid field '$field_name' (expected: $expected_type)` to stderr and exit 1.
       - Check these fields with type validation:
         - `id`: `.id | type == "string"`
         - `name`: `.name | type == "string"`
         - `description`: `.description | type == "string"`
         - `form`: `.form | type == "string"`
         - `version`: `.version | type == "string"`
         - `goals`: `.goals | type == "array" and length > 0`
         - `stages`: `.stages | type == "array" and length > 2`
         - `voice`: `.voice | type == "object"`
         - `voice.tone`: `.voice.tone | type == "string"`
         - `voice.formality`: `.voice.formality | type == "string"`
         - `voice.sentenceLength`: `.voice.sentenceLength | type == "string"`
         - `voice.paragraphStyle`: `.voice.paragraphStyle | type == "string"`
         - `voice.rhetoricalStyle`: `.voice.rhetoricalStyle | type == "string"`
         - `structure`: `.structure | type == "object"`
         - `structure.expectedSections`: `.structure.expectedSections | type == "array" and length > 0`
         - `structure.sectionOrder`: `.structure.sectionOrder | type == "string"`
         - `structure.paragraphPatterns`: `.structure.paragraphPatterns | type == "array" and length > 0`
         - `structure.introStyle`: `.structure.introStyle | type == "string"`
         - `structure.endingStyle`: `.structure.endingStyle | type == "string"`
         - `rubric`: `.rubric | type == "object"`
         - `rubric.criteria`: `.rubric.criteria | type == "array" and length > 0`
         - `rubric.passing_threshold`: `.rubric.passing_threshold | type == "number"`
         - `rubric.critical_criteria`: `.rubric.critical_criteria | type == "array" and length > 0`
         - `constraints`: `.constraints | type == "object"`
         - `constraints.no_citation_invention`: `.constraints.no_citation_invention == true`
         - `constraints.no_stance_shift`: `.constraints.no_stance_shift == true`
         - `transformations`: `.transformations | type == "object"`
         - `transformations.preserveVoice`: `.transformations.preserveVoice | type == "boolean"`
         - `examples`: `.examples | type == "array"`
       - After field checks, verify rubric weights sum to 1.0:
         ```bash
         WEIGHT_SUM=$(jq '[.rubric.criteria[].weight] | add' "$PRESET_FILE")
         if ! echo "$WEIGHT_SUM" | awk '{ if ($1 < 0.99 || $1 > 1.01) exit 1 }'; then
           echo "ERROR: rubric.criteria weights sum to $WEIGHT_SUM -- must sum to 1.0 (+/- 0.01)" >&2
           exit 1
         fi
         ```
       - Verify critical_criteria contains required entries:
         ```bash
         if ! jq -e '.rubric.critical_criteria | contains(["factual_integrity", "voice_preservation"])' "$PRESET_FILE" > /dev/null 2>&1; then
           echo "ERROR: rubric.critical_criteria must include 'factual_integrity' and 'voice_preservation'" >&2
           exit 1
         fi
         ```
       - Verify each rubric criterion has name, description, weight:
         ```bash
         INVALID=$(jq '[.rubric.criteria[] | select(.name == null or .description == null or .weight == null)] | length' "$PRESET_FILE")
         if [[ "$INVALID" -gt 0 ]]; then
           echo "ERROR: $INVALID rubric criteria missing required fields (name, description, weight)" >&2
           exit 1
         fi
         ```
       - On success: print `OK: $PRESET_FILE is valid` and exit 0.
       - Make the script executable: `chmod +x scripts/validate-preset.sh`
  </action>
